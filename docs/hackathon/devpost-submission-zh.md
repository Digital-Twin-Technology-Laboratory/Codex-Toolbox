# Devpost 提交文案（中文对照）

> Devpost 建议提交英文版；本文件用于你审阅叙事、核对事实，不必直接粘贴。

## 基本信息

- **项目名：** Show Codex IQ
- **标语：** Pick the right Codex model—without leaving your menu bar.
- **赛道：** Developer Tools
- **一句话介绍：** 一款原生 macOS 菜单栏应用，把 Codex 实时基准数据转化为智商、费用、耗时和个性化综合排名，让开发者一眼选出更合适的模型。

## Inspiration｜灵感来源

选择 Codex 模型并不是单一指标问题：智商最高的不一定最快，也不一定最省钱，而且最佳选择会随着最新测试结果变化。

Codex 雷达已经提供了有价值的快照，但反复打开浏览器、定位模型、比较三个指标会打断开发节奏。我希望把这个决策放到每天都会看到的位置——macOS 菜单栏。

项目由此聚焦到一个问题：**能不能把一次网页查询，缩短为两秒钟的一眼判断？**

## What it does｜它能做什么

- 菜单栏用紧凑的两行展示当前前两名。
- 可按智商、费用、耗时或个性化综合分排序。
- 展开面板集中展示四类榜单、数据新鲜度、手动刷新、Top 5 展开和本地趋势图。
- 三段式双滑块让智商、费用、耗时权重始终合计 100%，拖动后无需请求网络即可重新计算综合排名。
- 支持模型简称、序号样式、详细数值、左侧图标和趋势图开关，并根据内容自适应菜单栏宽度与面板高度。
- 启动时先展示最近一次成功快照；断网或刷新失败不会清空已有排名。
- 提供 macOS 14+ Universal 2 DMG，不含分析 SDK、广告、账号凭据或第三方运行时依赖。

## How we built it｜如何构建

项目使用 Swift 6、SwiftUI、AppKit、Swift Charts、Observation、URLSession 和 ServiceManagement。

数据层通过容错 `Codable` 模型解析 Codex 雷达公开快照。网络客户端支持 ETag、Last-Modified、200/304；actor 仓库将并发刷新合并为单次请求；本地存储原子保存最新成功快照和费用历史。UI 明确区分加载、刷新、陈旧和错误状态。

`RankingEngine` 是可独立测试的纯函数模块：智商降序，费用与耗时升序；综合榜将名次换算为百分位，使用平均名次处理并列，再套用用户权重和确定性的同分规则。

展开面板和设置使用 SwiftUI；菜单栏状态项使用 `NSStatusItem + NSHostingView`，以获得可靠的双行渲染和紧凑宽度。macOS 新版本使用原生 Liquid Glass，macOS 14–15 自动回退为系统材质。

### Build Week 范围说明

正式 Submission Period 开始前，项目已经存在一个包含 6 个提交的早期可运行原型；这部分必须明确列为赛前基线，不能冒充 Build Week 成果。赛期内，**Codex 与 GPT‑5.6** 对项目进行了实质扩展：实现三向权重控件、趋势图开关和自适应弹窗、菜单栏自适应宽度、模型简称、公开发布文档与更可靠的打包链路。后续任务根据截图和真实安装失败继续迭代，诊断 Library Validation、改为静态链接，并把“安装包能否真实启动”加入发布校验。

产品判断仍由人完成：解决什么问题、哪些取舍应该交给用户、菜单栏必须压缩到什么程度、哪些失败绝不能损坏现有体验。Codex 加速的是从判断到可运行、可验证版本的循环。

## Challenges we ran into｜遇到的挑战

1. **菜单栏不是普通窗口。** 纯 `MenuBarExtra` 无法稳定满足双行和极致紧凑宽度，最终只把状态项切到 AppKit，其余界面保留 SwiftUI。
2. **三项权重必须实时保持有效。** 系统没有原生三段双滑块控件，因此实现了带边界约束和辅助功能的自定义控件，始终保证合计 100%。
3. **实时数据不能导致脆弱体验。** 需要同时处理缺失字段、未知字段、304、超时、并发刷新、离线启动和旧数据保留。
4. **源码通过不等于安装包可用。** 一版 beta 因 ad-hoc 签名、Hardened Runtime 与动态库验证的组合而静默启动失败；最终改为静态链接，并把签名、架构、DMG 布局、校验值和真实启动纳入统一验证脚本。

## Accomplishments that we're proud of｜值得自豪的成果

- 交付了可安装、可运行、支持双架构的原生产品，而不是技术概念稿。
- 把智商、费用、耗时三维选择压缩成菜单栏上一眼可见的决策。
- 综合排名具备并列处理、缺失数据规则、确定性排序和实时自定义权重。
- 离线和失败场景不破坏已有答案，刷新请求会自动去重。
- 无第三方运行时依赖、无遥测、无广告、无账号凭据。
- 正式赛期内形成 9 个提交，涉及 87 个文件、2,576 行新增和 465 行删除，并产出 5 个 beta 安装包；赛前 6 个提交单独列为既有基线。

## What we learned｜学到了什么

原生菜单栏产品最合适的边界有时恰好位于两个框架之间：AppKit 负责精准的状态项行为，SwiftUI 负责更大的产品界面。

我们也认识到，“实时”数据只有在正确处理陈旧和部分状态时才值得信任。缓存、确定性排序和数据新鲜度提示，与图表本身同样重要。

Codex 在约束明确时发挥最好：例如“权重永远合计 100%”“刷新失败不能清空榜单”“DMG 必须真的启动”。人没有被移出开发过程，而是更集中地承担产品判断、约束定义与迭代验收。

## What's next｜下一步

扩大分发前，优先完成数据源授权复核以及 Developer ID 签名、公证。产品侧计划加入排名变化通知、更丰富的本地历史、对比导出、英文界面和更多无障碍测试。

长期目标是让排名层与数据源解耦，让开发者可接入自己信任的基准数据，同时保留这套快速、原生的模型选择体验。
