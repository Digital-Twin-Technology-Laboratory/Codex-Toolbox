# 额度估算与趋势范围设计

## 目标

Token 榜单要区分“只有 3 个根任务”与“展开失败”，并在不伪造官方 Token 上限的前提下，给出每项任务对账户额度的近似占比。模型趋势图默认不占用看板空间，由用户选择是否显示及时间范围。

## 数据与算法

`account/rateLimits/read` 的同一次只读响应同时提供额度窗口。应用仅解析并缓存 `windowDurationMins`、`usedPercent` 和 `resetsAt`，忽略 limit ID、凭据及其他账户字段。任务估算值为：

`taskTokens / localTokensInWindow × accountUsedPercent`

周窗口使用本机账本中相交日期的用量；5 小时窗口因账本只按日保存，是更粗粒度的近似值。界面因此始终标注“仅供参考”；账户未返回某个窗口时不构造数值。

## 交互与兼容

榜单标题始终显示当日实际根任务数。超过 3 项时整张卡片可在 Top 3 与设置的 Top 5/10 之间切换；不足时显示“今日仅 N 项”。新鲜安装默认关闭模型趋势图，已保存的开关值不被覆盖；趋势范围可选 7/14/30/90 天。重置卡缓存 schema 升级到 v3，仍能读取 v2，并在缺少额度窗口时立即安全刷新。
