# 额度估算与趋势范围设计

## 目标

Token 榜单要区分“只有 3 个根任务”与“展开失败”，并在不伪造官方 Token 上限或设备归属的前提下展示账户窗口用量。模型趋势图默认不占用看板空间，由用户选择是否显示及时间范围。

## 数据与算法

`account/rateLimits/read` 的同一次只读响应同时提供额度窗口。应用仅解析并缓存 `windowDurationMins`、`usedPercent` 和 `resetsAt`，忽略 limit ID、凭据及其他账户字段。

初版曾使用以下公式把账户用量分配给本机任务：

`taskTokens / localTokensInWindow × accountUsedPercent`

该公式假设账户窗口内的全部消耗都来自当前 Mac。账户同时在其他设备使用时，远端消耗只会进入 `accountUsedPercent`，不会进入本机分母，因而会被错误分摊给本机任务；本机窗口内只有一个任务时甚至会直接显示完整的账户百分比。5 小时窗口使用按日账本也无法满足时间粒度要求。

修订后的实现不再使用账户总比例分摊公式，而是解析 rollout 每个 `token_count` 事件自带的额度快照。应用把同一窗口内的本机事件按时间合并，超过 15 分钟无活动就切断归因；连续区段内观察到的额度百分点按各根任务 Token 分配，并以本机多个干净区段的 Token/百分点中位数补足首尾不足 1% 的取整部分。账户没有返回窗口、缓存窗口已过期或快照不足时不构造任务数值。

任务行使用 `≈` 标记估算，并提供中/低置信度提示；有效账户窗口仍在卡片说明中单独显示为“账户总用量（含所有设备）”。其他设备与本机同时工作的区段仍可能影响估算，因此该结果不是官方按设备或任务拆分。

## 交互与兼容

榜单标题始终显示当日实际根任务数。超过 3 项时整张卡片可在 Top 3 与设置的 Top 5/10 之间切换；不足时显示“今日仅 N 项”。新鲜安装默认关闭模型趋势图，已保存的开关值不被覆盖；趋势范围可选 7/14/30/90 天。重置卡缓存 schema 升级到 v3，仍能读取 v2，并在缺少额度窗口时立即安全刷新。
